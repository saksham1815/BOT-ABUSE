============================================================
RULE ENGINE PROPERTIES
============================================================

Types with a ? suffix (e.g. bool?) can be missing (nil).
You may want to check the values with name? before reading them.
Reading a property which is nil invalidates the whole rule,
meaning it will not trigger.

============================================================
1.1 MISC
============================================================

access_time : datetime
Description:
    The time when the request was received
Example:
    2020-01-15T21:24:16.789Z

ai_category : string?
Description:
    AI tool category if the request comes from AI tool.
    Can only be set for CWAF domains.

ai_tool : string?
Description:
    Name of the AI tool if the request comes from such.
    Can only be set for CWAF domains.

apollo : set<ident>
Description:
    Flags set by Apollo (machine learning)

apollo.generating_uas_mismatch : bool
apollo.ip_id_ratio : bool
apollo.no_gen_requests : bool

client_platform : string?
Description:
    The platform the visitor is using.
Example:
    "web_browser"

cloudwaf_website_id : number?
Description:
    Current domain CloudWAF website ID

cookie_list : array<struct>

    List of all cookies, in order, with correct case-sensitive names

============================================================
1.2 FLAGS
============================================================

Flags are all booleans.
They are always true or false, never missing.

flags
Description:
    New flags joined with previous token flags

flags.automation_chrome_driver
Description:
    Automation detected through ChromeDriver

flags.jsat_user_agent_header_inconsistent
Description:
    User-Agent mismatch detected

flags.no_token
Description:
    Visitor lacks token

flags.suspicious_user_agent
Description:
    User-Agent is suspicious or malformed
